# Interview Coding Prep

A quick 3-day refresh plan for an upcoming coding interview, built as a reusable system rather than ad-hoc cramming.

The premise: I've been practicing LeetCode consistently over the past few months (~130 problems at [SansWord/leetcode_submissions](https://github.com/SansWord/leetcode_submissions)), so the patterns are mostly in place. Rather than grinding random problems before the interview, I directed Claude to mine my own submission archive — identify which patterns I already have clean reps for, flag the gaps, and generate a refresh doc keyed to my specific prior solutions. That refresh doc drives a tight problem list, and a disciplined pair-programming loop handles the parts of prep where AI tooling helps.

## What's in this repo

| File | Purpose |
|---|---|
| [`CLAUDE.md`](CLAUDE.md) | Active prompt for this repo — **practice mode**. Scoped to drilling problems from `PLAN.md` against patterns in `refresh_knowledge.md`, writing solutions into `solutions/`. Tuned for known Python / known structure. |
| [`CLAUDE.pair_programming.md`](CLAUDE.pair_programming.md) | **Portable** prompt for live pair-programming in an unfamiliar codebase. Auto-detects language, gathers context from surrounding code before proposing, respects existing conventions, and enforces a **commit-only-when-I-approve** git workflow. Copy into any working directory as `CLAUDE.md` to activate. |
| [`refresh_knowledge.md`](refresh_knowledge.md) | Pattern library keyed to my own past solutions at [SansWord/leetcode_submissions](https://github.com/SansWord/leetcode_submissions). Each section cites a specific prior solve, extracts the invariant, and gives a clean template. Plus a section on patterns that need fresh reps. |
| [`PLAN.md`](PLAN.md) | Daily schedule, 15-problem core list, 3 design-flavored problems for AI pair-programming practice, 15-problem stretch list, session log. |
| [`solutions/_template.py`](solutions/_template.py) | Per-problem scaffold: header block for pattern/time tracking, inline `assert` tests covering happy path / empty / single / duplicates / one adversarial case. |

Actual solutions live in `solutions/` and are gitignored — the repo shares the *system*, not the answers.

## How the AI loop works

The pattern refresh doc was generated by Claude analyzing the [leetcode_submissions](https://github.com/SansWord/leetcode_submissions) archive: reading representative solutions, identifying my idioms, extracting invariants I already use intuitively, and flagging gaps where I have no recent reps. Every pattern section in `refresh_knowledge.md` links back to a specific file in that repo — so the claims are verifiable, not just assertions.

During problems, `CLAUDE.md` forces Claude to propose approaches before coding, wait for my pick, and surface its reasoning. I read every line and push back at least once per problem. That's the demonstration: AI as pair programmer under explicit direction, not autocomplete.

## Reusing this

The template at `solutions/_template.py` and the workflow in `CLAUDE.md` are problem-agnostic. `refresh_knowledge.md` is keyed to my submissions, but the same generation approach — "analyze my archive, build a refresh doc citing my own prior work" — generalizes to any practice corpus.
